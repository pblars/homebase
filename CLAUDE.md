# Home Base

A full-screen family home dashboard for a Surface tablet mounted in the house.
Single deployable project, **vanilla HTML/CSS/JS ‚Äî no framework, no build step**.
Hosted on Cloudflare Pages.

- GitHub: `pblars/homebase` (`git@github.com:pblars/homebase.git`)
- Weather: OpenWeatherMap (free tier), zip `37064` ‚Äî Franklin, TN
- Calendar: Google Calendar API (later phase)

---

## Architecture overview

The app is a stack of full-bleed layers inside a locked, `overflow:hidden`
viewport:

```
 z-index
  10   #screen-root      ‚Üê screens mount here (Phase 2+: Dashboard, Quest, etc.)
   0   #sky-stage
         ‚îú‚îÄ‚îÄ #sky-a      ‚Üê sky image layer A  (crossfaded)
         ‚îî‚îÄ‚îÄ #sky-b      ‚Üê sky image layer B  (crossfaded)
       (body background #0a0f0d shows before any image loads)
  off   #sky-preload     ‚Üê off-screen <img>, warms next slot's art into cache
```

### The sky system (Phase 1 ‚Äî the only thing built so far)

Three singletons, wired together in `src/main.js`:

| Module                       | Responsibility                                             |
|------------------------------|------------------------------------------------------------|
| `src/sky/TimeSystem.js`      | Maps the clock to one of 8 **time slots**; fires on change |
| `src/sky/WeatherSystem.js`   | Fetches OWM weather/forecast; maps codes to a **condition**|
| `src/sky/SkyManager.js`      | Resolves `(slot, condition)` ‚Üí image, **crossfades** layers|

Data flow:

```
TimeSystem.onSlotChange ‚îÄ‚îê
                         ‚îú‚îÄ‚ñ∫ main.js syncSky() ‚îÄ‚ñ∫ SkyManager.update(slot, condition)
WeatherSystem.onUpdate ‚îÄ‚îÄ‚îò                                   ‚îÇ
                                                             ‚ñº
                                            resolveKey() ‚Üí crossfadeTo(image)
```

`SkyManager` only crossfades when the **resolved image key actually changes**
(slot change OR weather condition change) ‚Äî never every tick. Crossfades run for
2.5s via CSS opacity transitions. When a slot change is ‚â§ 5 minutes away, the next
image is preloaded off-screen so the transition is instant.

## Sky image naming convention  ‚Üê READ THIS BEFORE ADDING ART

Drop illustrated images into **`/assets/sky/`** named exactly:

```
{slot}_{condition}.{webp|png}
```

**Slots** (8): `night` `predawn` `dawn` `morning` `midday` `afternoon` `golden` `dusk` `evening`

**Conditions**: `sunny` `clear` `cloudy` `rainy` `stormy` `foggy` `snowy` `partly_cloudy`

Examples: `golden_sunny.webp`, `night_clear.png`, `midday_stormy.webp`

**Format:** `.webp` and `.png` both work. `SkyManager` tries `.webp` first, then
`.png` (see `EXTENSIONS` in `src/sky/SkyManager.js`). WebP is preferred ‚Äî much
smaller files mean faster loads and smoother crossfades on the wall tablet ‚Äî but
PNGs are fine to start with. You can mix formats; if both `foo.webp` and
`foo.png` exist, the `.webp` wins.

**No restart needed.** Just drop the file in. `SkyManager` re-resolves on the next
slot change or weather refresh and crossfades it in. (To see it instantly, reload.)

### Which 27 images ship in the design

These are the keys `SkyManager` knows about (its hardcoded availability map). The
art for these is what you generate (e.g. via DALL¬∑E) and drop in:

```
night:     clear  cloudy  rainy  snowy
predawn:   clear  cloudy
dawn:      clear  cloudy  rainy
morning:   sunny  cloudy  rainy
midday:    sunny  cloudy  stormy  rainy
afternoon: sunny  cloudy  stormy  foggy
golden:    sunny  cloudy  rainy
dusk:      clear  cloudy
evening:   clear  cloudy
```
Plus `placeholder.webp` (the dark fallback).

> If you add a brand-new key that isn't in the list above, also add it to the
> `AVAILABLE` set in `src/sky/SkyManager.js` so the resolver will pick it.

### How fallback resolution works

`resolveKey(slot, condition)` walks this chain until it finds a file that exists:

1. **Exact**: `{slot}_{condition}` (e.g. `golden_rainy`)
2. **Condition synonyms / degradation** in the same slot
   (`sunny`‚Üî`clear`, `partly_cloudy`‚Üí`cloudy`, `rainy`/`stormy`/`snowy`/`foggy`‚Üí`cloudy`)
3. **Generic same-slot** fallback: `cloudy`, then `sunny`/`clear`
4. **Adjacent slot**, same condition chain (slots wrap: `night`‚Üî`evening`)
5. **`placeholder`**

So a missing image never breaks the screen ‚Äî it degrades to the nearest sensible
art, and ultimately the dark placeholder.

## The placeholder image

`assets/sky/placeholder.webp` is a **solid `#0a1a0f` dark green-black** 1√ó1
lossless WebP, generated by `scripts/gen-placeholder.js` (no image libraries
needed ‚Äî it hand-encodes a minimal VP8L bitstream). Because the sky `<img>` uses
`object-fit:cover`, the 1√ó1 pixel stretches to fill the screen, so it looks like a
full solid rectangle. Real illustrated images replace it per the naming
convention above.

Regenerate (or change the color) with:

```
node scripts/gen-placeholder.js
```

## Config setup

1. Copy the example config and fill in real values:
   ```
   cp config.example.js config.js
   ```
2. Edit `config.js` ‚Äî add your `OPENWEATHERMAP_API_KEY` (and later Google keys).
3. **`config.js` is gitignored ‚Äî never commit it.** `config.example.js` documents
   the shape and is the file that's committed.

`config.js` is loaded by a plain `<script src="config.js">` tag in `index.html`
**before** all other scripts, so `CONFIG` is a global available everywhere.

Keys:

| Key                       | Purpose                                  |
|---------------------------|------------------------------------------|
| `OPENWEATHERMAP_API_KEY`  | Live weather + forecast                  |
| `GOOGLE_CALENDAR_ID`      | Calendar events (later phase)            |
| `GOOGLE_API_KEY`          | Google Calendar API auth (later phase)   |
| `LAT` / `LON`             | 35.9251 / -86.8689                       |
| `ZIP`                     | `37064`                                  |
| `LOCATION_LABEL`          | `Franklin, TN`                           |

## Running locally

It's static ‚Äî serve the folder over HTTP (don't open via `file://`, the relative
script/asset paths and `fetch` need an origin):

```
# any static server works, e.g.:
npx serve .
# or
python -m http.server 8080
```

Then open the served URL.

## Deploy

```
npx wrangler pages deploy . --project-name homebase
```

### Handy console commands for testing

```js
// What's showing now?
SkyManager.currentKey

// Resolve any combination without changing the screen
SkyManager.resolveKey('midday', 'stormy')   // -> "midday_stormy"
SkyManager.resolveKey('predawn', 'snowy')   // -> falls back

// Force a crossfade to a specific slot/condition
SkyManager.update('night', 'clear')

// Re-fetch weather immediately
WeatherSystem.refresh()
```
